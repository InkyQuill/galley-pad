# Stage 0 Project Definition Plan

## Goal

Create enough documentation to start implementation without allowing the app scope to drift into a workspace product.

## Outputs

- `README.md`
- `docs/vision.md`
- `docs/product-principles.md`
- `docs/architecture.md`
- `docs/roadmap.md`
- `docs/reference/galley-editor.md`
- staged plans in `docs/plans/`

## Steps

1. Write the product vision.
2. Define strict product principles and non-goals.
3. Document the Tauri, React, and Galley Editor architecture.
4. Write a staged roadmap from scaffold to daily-driver packaging.
5. Capture Galley Editor installation and integration assumptions.
6. Review all docs for scope creep, contradictions, and vague placeholders.

## Exit Criteria

- The first implementation stage can begin without further product discovery.
- The core product promise is repeated consistently across docs.
- Future features are separated from v1 requirements.
